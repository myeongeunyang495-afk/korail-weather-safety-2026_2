## 1. 배경 · 목표

- **원본 서비스**: 옥외 철도근로자(선로유지보수·입환·건설공사)의 **온열·한랭질환** 예방 도구. 산업안전보건법 개정(’25.7.17)으로 체감온도 관리가 사실상 의무화된 배경에서 수도권서부본부가 자체 개발.
- **이번 주문**: ① **전사(전국) 확대 사용**을 위한 페이지 생성 ② 사용자에게 유용한 **추가 정보·서비스 제안·반영** ③ 가능하면 **동일한 웹 방식**으로 구현 ④ **git/GitHub 저장소** 생성 ⑤ **플랜 컨펌 후** 구현.
- **원본 PDF가 직접 밝힌 향후 로드맵과 일치**: 전사 확대 / 한파·호우 등 계절 확대 / 상황보고 초안 자동생성 / 정량 성과 데이터화.

### 2.1 확보한 체감온도 공식(여름, 기상청 2022)
```
Tw(습구온도, Stull) =
  T·atan(0.151977·√(RH+8.313659)) + atan(T+RH)
  − atan(RH−1.67633) + 0.00391838·RH^1.5·atan(0.023101·RH) − 4.686035
체감온도 = −0.2442 + 0.55399·Tw + 0.45535·T − 0.0022·Tw² + 0.00278·Tw·T + 3.0
```
(T=기온℃, RH=상대습도%) — 기상청 격자변환(DFS Lambert)·base_time 로직도 원본 코드로 확보.

## 3. 확장 범위

### 3.1 전사(전국) 확대
- 지역 선택을 **전국 17개 시·도 → 시·군·구 → 읍·면·동**으로 확장.
- **데이터 출처**: 기상청 「단기예보 오픈API 활용가이드」 **격자좌표 엑셀**(행정동→nx,ny,경위도) 기반 **전국 지역 DB 자체 생성** → 지역선택에 Kakao 의존 제거(좌표 내장). GPS 표시명만 Kakao(프록시 경유).
- 본부/부서 무관 **누구나** 쓰는 **전사 소개·온보딩 섹션**(서비스 소개 + QR + 홈화면 추가 안내) 포함.

### 3.2 계절 통합(폭염 + 한파 + 호우) — 사용자 선택 반영
- **여름 체감온도**(상기 공식) + **겨울 체감온도(풍속 기반 wind chill)**:
  `13.12 + 0.6215·T − 11.37·V^0.16 + 0.3965·V^0.16·T` (V=풍속 km/h). 기상청 겨울 공식.
- **호우**: 기상청 호우 주의보/경보 연계 + 강수 안전수칙.
- 계절 **자동 판별(월·기온 기반) + 수동 토글**. 프록시가 **풍속(WSD)·강수(RN1·PTY)** 추가 수집.

### 3.3 추가 기능(사용자 4종 전부 선택 반영)
1. **기록일지 자동화** — 체감온도 31℃↑(또는 한파 임계↓) 시 법정 「체감온도 기록일지」 자동 생성(시각·위치·기온·습도·체감온도·단계·조치) → 목록·**CSV/인쇄(PDF)** 내보내기. localStorage 저장(서버 동기화는 Phase 2).
2. **시간대별 예보 + 작업시간 권고** — 초단기예보+단기예보로 향후 6~12h 체감온도 그래프, 위험 시간대(예: 14~17시) 하이라이트·작업 회피 권고.
3. **응급조치 가이드 + 현장 공유** — 온열/한랭질환 증상·응급조치 카드 + 현재 상태를 **Web Share/링크/QR**로 팀에 즉시 전파.
4. (3에 포함) **즐겨찾는 현장** — 자주 보는 작업장소 저장 후 원터치 조회.

### 3.4 v1 제외(Phase 2 후보)
- 관리자 대시보드(본부·사업장 집계), 서버 DB 기록 동기화, 상황보고 **자동 전송**(v1은 초안 생성까지), 다국어.

## 4. 아키텍처

```
[모바일 브라우저 / PWA]  React 18 + Vite 5 + TypeScript + vite-plugin-pwa
  ├─ 체감온도 엔진(여름/겨울)  ├─ 단계 엔진(계절별)  ├─ 격자변환/base_time
  ├─ WeatherProvider 추상화 ── (a) KMA 프록시 provider(운영)
  │                            (b) Mock provider(키 없이 데모/개발)
  ├─ localStorage: 기록일지·즐겨찾기·최근위치
  └─ Service Worker(오프라인 셸 + 가이드 캐시)
        │
        ▼
[서버리스 프록시]  Netlify Functions(이식 가능)
  ├─ /api/weather?lat&lon → 기상청 초단기실황+예보(T1H,REH,WSD,RN1,PTY) + 격자/base
  └─ /api/geocode        → Kakao 프록시(키 은닉)
        │
        ▼
[기상청 공공데이터포털 API] + [Kakao Local API]   (키는 환경변수)
```
- **개선점**: 풍속·강수 추가 / Kakao 키 프록시化(노출 제거) / 전국 지역 DB 내장 / 계절 자동전환 / 기록·즐겨찾기 영속화.

## 5. 단계 시스템(계절별)

- **폭염(체감온도↑)**: 정상<31 / 관심≥31 / 주의보≥33 / 경보≥35 — **KORAIL 기존 수칙 그대로 유지(검증된 공식 문구)**.
- **한파(체감온도↓)**: 기상청 한파 체감온도 기준 + KOSHA 한랭질환 예방수칙으로 4단계 정의. → **문구는 KORAIL 안전보건처 검토 필요(초안 제공)**.
- **호우**: 기상청 호우 주의보/경보 + 감전·미끄럼·시야·하천 안전수칙(초안). → 동일하게 **검토 필요**.

## 6. 디자인 방향 (KORAIL 블루 신뢰형)

- **브랜드**: KORAIL 코퍼레이트 블루 기본(정확값은 로고에서 추출) + 흰 서피스. **4단계 안전색(정상=초록·관심=노랑·주의보=주황·경보=빨강) 유지**(현장 인지성).
- **타이포**: Pretendard **오프라인 임베드**(사내망 호환). 큰 체감온도 수치, 명확한 단계 배너/팝업.
- **현장성**: 큰 탭 타깃(장갑 대응), 고대비 옵션, 모바일 우선. 원본 UI는 베끼지 않고 새로 브랜딩.

## 7. 기술 스택

- React 18 + Vite 5 + TypeScript + `vite-plugin-pwa`(Workbox).
- 무거운 UI 라이브러리 미사용 — CSS 토큰/모듈. 예보 그래프는 경량 SVG/Canvas. QR은 경량 라이브러리.
- 서버리스: Netlify Functions(Node). Pretendard 셀프호스팅. 테스트: Vitest(엔진) + Playwright(시각/접근성).
