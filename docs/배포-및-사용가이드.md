# 배포 · 운영 · 사용 가이드

## 1. 로컬 개발

```bash
npm install
npm run dev      # http://localhost:5173 (기본 Mock 데이터로 즉시 동작)
npm test         # 엔진 단위테스트
npm run build    # 타입체크 + 프로덕션 빌드(dist) + PWA 생성
npm run preview  # 빌드 결과 미리보기
```

키가 없어도 **Mock provider**로 전 기능이 동작한다. 데모 시 특정 단계를 강제하려면 쿼리스트링:

```
?demo=heat | warning | interest | danger | cold | rain
```

## 2. 실데이터(기상청) 연동

### 2.1 키 발급
- **기상청**: [공공데이터포털](https://www.data.go.kr) → “기상청_단기예보 ((구)_동네예보) 조회서비스” 활용신청 → 일반 인증키(Decoding) 확보 → `KMA_SERVICE_KEY`.
- **Kakao**(GPS→행정동명, 선택): [Kakao Developers](https://developers.kakao.com) → 앱 생성 → REST API 키 → `KAKAO_REST_KEY`. (없으면 내장 시군구 최근접명으로 표시)

### 2.2 키 주입
서버리스 함수에서만 **환경변수**로 사용한다(코드/깃에 키를 넣지 않는다). `.env.example` 참고.
- 키가 설정되면 `/api/weather`가 실데이터를 반환하고, 미설정 시 503 → 클라이언트가 자동으로 Mock 폴백.

## 3. 배포

### 3.1 Netlify (권장 데모)
1. 본 저장소를 Netlify에 연결(또는 `netlify deploy`).
2. Site settings → Environment variables 에 `KMA_SERVICE_KEY`, `KAKAO_REST_KEY` 등록.
3. `netlify.toml`이 빌드(`dist`)·함수(`netlify/functions`)·`/api/*` 리다이렉트를 자동 구성.

### 3.2 사내/타 호스팅 이식
- 프론트는 정적(`dist`) → 어떤 정적 웹서버에도 배포 가능.
- 프록시(`/api/weather`, `/api/geocode`)는 표준 fetch 기반 → Cloudflare Workers, Vercel Functions, 사내 Node 서버 등으로 이식 가능(엔드포인트 경로만 맞추면 됨).
- **오프라인/사내망**: Pretendard 폰트는 자체 호스팅(`public/fonts`), PWA가 셸·가이드·폰트를 캐시한다.

## 4. 현장 사용 매뉴얼

| 단계 | 사용법 |
|---|---|
| 1 | QR 스캔 또는 링크로 접속 → 브라우저 메뉴 **‘홈 화면에 추가’**(앱처럼 사용) |
| 2 | **‘내 위치 실시간 조회’**(GPS) 또는 **지역 선택**으로 현재 체감온도 확인 |
| 3 | 단계 팝업의 조치사항 확인 → **‘현장 확인 및 조치 완료’** = 기록일지 자동 저장(관심 단계↑) |
| 4 | **예보** 탭에서 위험 시간대 회피 작업계획, **안전가이드**에서 응급조치 확인 |
| 5 | **기록일지** 탭에서 CSV 내보내기/인쇄(법정 보관), **현장 공유**로 팀 전파 |

## 5. 운영 주의

- **폭염** 조치문구 = KORAIL 원본의 검증된 표준 문구.
- **한파·호우** 조치문구·임계값 = KOSHA·기상청 기준 **초안(draft)**. 정식 배포 전 **안전보건처 검토·확정** 필요(앱 내 ‘초안’ 표시).
- 기록일지는 현재 단말 `localStorage` 저장 → 법정 보관을 위해 **CSV 내보내기 권장**. (서버 동기화는 Phase 2)
